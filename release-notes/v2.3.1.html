Version 2.3.1

Released on : 19/12/2025
Operational version

Features​

Support for Postgres SSL connections​

REGARDS can now be configured to connect to Postgres databases using SSL connections.
It is also possible to separately configure the database details for the microservice rs-processing.

Breaking change​

Deployment​

SSL connection to the database​

SSL connection to the database can now be controlled in the inventory, using the ssl.mode option. This option can be added under the following section in the inventory's group_vars/regards_nodes/main.yml file :

group_config_mservices:
  postgres:
    instance:
      ssl:
        mode: require

More information about this ssl option is available in the Postgres database setup section.

Database connection for rs-processing​

By default, the microservice rs-processing stores its data into the instance database. It is now possible to use a different database for this service. For more details, check out the Postgres database setup section.

Internal database​

If REGARDS internal postgres instance is used, SSL in database connections can be enabled by editing the inventory's group_vars/regards_nodes/main.yml file :

group_docker_cots:
  postgres:
    ssl: true

Bug fixes​

  • rs-dam

    • Fixes an issue concerning the updating access rights with dynamic plugins (example: IDataObjectAccessFilterPlugin interface) in the REGARDS catalog
    • Fixes an issue concerning the adding of Markdown file in dataset (only last version Firefox)

  • rs-frontend

    • Fixes an issue concerning the map displaying in HMI REGARDS user

  • rs-storage

    • Fixes an issue concerning the STAF plugin with filenames containing special character.

A framework and a set of plugins to build scientific archives.

REGARDS is an Open Source data valorization product, highly flexible to various themes, modular, efficient, long-lasting, evolutionary and scalable, aims at providing services to Mission Centers and laboratories, and is able to deal with huge amount of data.
Our goal is to allow a strong commitment from user community in the development of modules and plugins.

We welcome your contributions to make these documents better.
