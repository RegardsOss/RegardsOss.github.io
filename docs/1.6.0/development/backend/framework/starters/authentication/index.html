Authentication starter

Purposeâ€‹

This starter is used to retrieve current user properties regardless of the authentication management.

Configurationâ€‹

Add starter dependency to your POM (version depends on the REGARDS BOM)

<dependency>
  <groupId>fr.cnes.regards.framework</groupId>
  <artifactId>authentication-regards-starter</artifactId>
</dependency>

Business dependency

<dependency>
  <groupId>fr.cnes.regards.framework</groupId>
  <artifactId>authentication-regards</artifactId>
</dependency>

Dependency :

  • None

Used by :

  • Security starter

Autoconfigurationâ€‹

Starter autoconfigures beans unless it already exists :

  • IAuthenticationResolver to retrieve current user and role at runtime,
danger

In production, the implementation of IAuthenticationResolver must be thread safe.

How toâ€‹

How to useâ€‹

Just inject bean in your component.

How to override default behaviourâ€‹

Create your own IAuthenticationResolver bean to implement your own authentication info retrieval.

A framework and a set of plugins to build scientific archives.

REGARDS is an Open Source data valorization product, highly flexible to various themes, modular, efficient, long-lasting, evolutionary and scalable, aims at providing services to Mission Centers and laboratories, and is able to deal with huge amount of data.
Our goal is to allow a strong commitment from user community in the development of modules and plugins.

We welcome your contributions to make these documents better.
