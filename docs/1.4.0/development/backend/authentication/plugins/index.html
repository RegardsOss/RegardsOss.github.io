Authentication plugins

Overviewâ€‹

note

This extension point allows to define an authentication protocol.

REGARDS provides many implementation of this extension point :

  • RegardsInternalAuthenticationPlugin : Authenticate users with internal managed accounts.
  • LdapAuthenticationPlugin : Authenticate users through a configured LDAP server

Interfaceâ€‹

IAuthenticationPlugin

Implementationâ€‹

To learn more about how to create your own plugin see Plugins

Here under is an exemple of how to implements this extension point to create your own business logic.

NOTE : The only mandatory meta data on every authenticated users is an email adress. So each authentication protocol must be able to return email of authenticated users.
@Plugin(id = "exemple", version = "1.0.0", description = "exemple plugin",
        author = "REGARDS Team", contact = "regards@c-s.fr", licence = "LGPLv3.0", owner = "CSSI",
        url = "https://github.com/RegardsOss")
public class Plugin implements IDataObjectAccessFilterPlugin {
   
   @Override
   public AuthenticationPluginResponse authenticate(String userName, String userPassword, String scope) {
           // Do authentication and return an AuthenticationPluginResponse
           Boolean accessGranted = true;
           String email = userName;
           return new AuthenticationPluginResponse(accessGranted,email);
   }
}

A framework and a set of plugins to build scientific archives.

REGARDS is an Open Source data valorization product, highly flexible to various themes, modular, efficient, long-lasting, evolutionary and scalable, aims at providing services to Mission Centers and laboratories, and is able to deal with huge amount of data.
Our goal is to allow a strong commitment from user community in the development of modules and plugins.

We welcome your contributions to make these documents better.
