## Features

### Support for Postgres SSL connections

REGARDS can now be configured to connect to Postgres databases using SSL connections.  
It is also possible to separately configure the database details for the microservice `rs-processing`.

## Breaking change

### Deployment

#### SSL connection to the database

SSL connection to the database can now be controlled in the inventory, using the `ssl.mode` option. This option can
be added under the following section in the inventory's `group_vars/regards_nodes/main.yml` file :

```
group_config_mservices:
  postgres:
    instance:
      ssl:
        mode: require
```

More information about this `ssl` option is available in the
[Postgres database setup section](../docs/next/setup/swarm/advanced/postgres-database).

#### Database connection for `rs-processing`

By default, the microservice `rs-processing` stores its data into the instance database. It is now possible to use a
different database for this service. For more details, check out the
[Postgres database setup section](../docs/next/setup/swarm/advanced/postgres-database).

#### Internal database

If REGARDS internal postgres instance is used, SSL in database connections can be enabled by editing the inventory's
`group_vars/regards_nodes/main.yml` file :

```
group_docker_cots:
  postgres:
    ssl: true
```
