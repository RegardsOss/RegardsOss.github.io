Version 1.4.0

disclaimer

This version modify some APIs of existing services.
Existing catalogs needs to be reset.

Manual migration#

As 1.4.0 Version of REGARDS modify the way to index any data in Elasticsearch catalog, catalog of every projects of a REGARDS instance should be reset. To do so, you can use the reset option from the administration interface as describe here

AMQP subsribers changes#

The here under changes modify how applications can interract with "Feature Manager" REGARDS sevice to send feautre requests through AMQP messages

AMQP Feature requests

Feature creation request

disclaimer

No changes in amqp interfaces.

Feature creation request by file reference

  • Reference requests in previous version are renamed to "Extraction requests" in V1.4.0.
  • New extraction requests has to be published on this exchange :
    • regards.broadcast.fr.cnes.regards.modules.featureprovider.domain.FeatureExtractionRequestEvent`
  • In previous version reference requests were sent to :
    • regards.broadcast.fr.cnes.regards.modules.feature.dto.event.in.FeatureReferenceRequestEvent
disclaimer

The requests body and header has not changed.

Feature patch request

disclaimer

No changes in amqp interfaces.

Feature deletion request

disclaimer

No changes in amqp interfaces.

AMQP monitoring API

Requests response

  • In previuous version requests responses were all sent to :
    • regards.broadcast.fr.cnes.regards.modules.feature.dto.event.out.FeatureRequestEvent
  • In new version All Creation/Notification/Deletion/Update requests responses are sent to :
    • regards.broadcast.fr.cnes.regards.modules.feature.dto.event.out.FeatureRequestEvent
  • In new version All Extraction requests responses are sent to :
    • regards.broadcast.fr.cnes.regards.modules.featureprovider.domain.FeatureExtractionResponseEvent

A framework and a set of plugins to build scientific archives.

REGARDS is an Open Source data valorization product, highly flexible to various themes, modular, efficient, long-lasting, evolutionary and scalable, aims at providing services to Mission Centers and laboratories, and is able to deal with huge amount of data.
Our goal is to allow a strong commitment from user community in the development of modules and plugins.

We welcome your contributions to make these documents better.
